# Security Policy

## Supported Versions

Security fixes are only guaranteed for the latest maintained code on the default
branch and the latest tagged release.

| Version        | Supported |
| -------------- | --------- |
| `main`         | Yes       |
| Latest release | Yes       |
| Older releases | No        |

## Reporting a Vulnerability

Please do not open a public GitHub issue for security vulnerabilities.

Report privately to:

- Email: `namelesslx@foxmail.com`
- Subject: `[codeless-blog security] <short summary>`

When possible, include:

- affected area or file path
- reproduction steps or proof of concept
- impact assessment
- suggested remediation, if you have one

Expected response targets:

- acknowledgement within 72 hours
- initial triage within 7 days
- coordinated disclosure plan after the issue is confirmed
