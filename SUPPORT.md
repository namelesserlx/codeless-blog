# Support

## Where to ask for help

- Bug reports: open a GitHub issue with a reproducible example
- Feature requests: open a GitHub issue with the use case and expected outcome
- Private security concerns: follow [`SECURITY.md`](./SECURITY.md)

## Before opening an issue

- Read the root [`README.md`](./README.md) and the English version if needed
- Confirm the problem is still reproducible on the latest code
- Include environment details, screenshots, logs, and exact commands when relevant

## Response expectations

This is an open-source project maintained in spare time, so response times may
vary. Clear, minimal, reproducible reports will always get the fastest traction.
