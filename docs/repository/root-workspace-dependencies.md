# 根工作区依赖分类清单

本文件用于说明根工作区 `package.json` 的依赖边界，目标是让根包维持“工程编排层”定位，而不是继续承接应用运行时依赖。

## 1. 根工作区应该保留什么

根工作区主要保留这些依赖：

- Monorepo 编排工具
- 代码规范与提交门禁工具
- 只服务仓库根层命令的工程依赖

当前保留在根层是合理的类型包括：

- `turbo`
- `pnpm` 对应的工作区脚本依赖
- `husky`
- `lint-staged`
- `commitlint`
- 根级 `eslint` / `prettier` / TypeScript 工具链

## 2. 当前已从根层移除的运行时依赖

以下依赖已经从根 `package.json` 中移除：

- `react`
- `react-dom`
- `axios`
- `dayjs`
- `@prisma/client`

移除原因：

- 根工作区没有直接消费这些运行时代码
- 它们已经在具体应用或共享包中各自声明
- 留在根层会放大版本漂移，也容易误导后续维护者认为根包本身参与运行时逻辑

## 3. 当前分类原则

- 应用运行时依赖：声明在应用自己的 `package.json`
- 共享包运行时依赖：声明在共享包自己的 `package.json`
- 工程编排依赖：声明在根工作区

## 4. 后续收敛方向

- 后续新增依赖时，先判断它属于应用、共享包还是根层工程依赖
- 若依赖只服务单一应用，不应提升到根工作区
- 若依赖只服务某个共享包，也不应放到根工作区兜底
