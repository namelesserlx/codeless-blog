# Blog 上线前功能清单

## 1. 文档目标

本清单用于梳理博客项目正式上线前的功能范围、优先级、验收标准与交付节奏，避免遗漏关键能力。

适用范围：

- 前台（`apps/blog`）
- 管理台（`apps/admin/client`）
- 管理端服务（`apps/admin/server`）

更新时间：2026-03-10

---

## 2. 上线目标（MVP）

- 管理端 Dashboard 可用，且展示真实数据。
- 前台首页无假数据，核心模块内容真实可点击。
- 前台补齐 About 与 Contact 页面。
- 关键页面具备基础错误处理与空状态。

## 4. 详细清单（可执行）

### 4.1 管理端 Dashboard（P0）

#### ADM-01 核心统计卡片

- 目标：
    - 展示文章总数、已发布、草稿、未发布。
    - 展示片段总数、已发布、草稿、未发布。
    - 展示评论总数与待审核数。
    - 展示浏览量（可先做近 7 天总量）。
- 依赖：
    - 复用现有 `article stats`。
    - 补充 `snippet stats`。
    - 补充/增强 `comment stats`（返回状态分布）。
- 验收标准：
    - 页面无硬编码数字。
    - 接口异常时有错误提示和重试入口。

#### ADM-02 趋势图（7/30 天）

- 目标：
    - 提供文章浏览、片段浏览趋势图（按天）。
    - 支持切换时间范围（7 天/30 天）。
- 依赖：
    - 新增聚合接口，按日期返回序列数据。
- 验收标准：
    - 切换时间维度时图表正确刷新。
    - 空数据时展示空状态。

#### ADM-03 待处理事项

- 目标：
    - 展示待审核评论 Top N。
    - 展示草稿文章 Top N。
- 验收标准：
    - 每条记录可跳转到对应管理页并定位处理。

---

### 4.2 前台首页真实化（P0）

#### WEB-01 Hero 统计真实化

- 目标：
    - 将“文章数/阅读量/关注者”等硬编码替换为真实数据。
    - 暂无“关注者”真实来源时，改为可真实统计的指标（如标签数、片段数）。
- 验收标准：
    - 首屏所有统计来源可追溯，文案与数据一致。

#### WEB-02 热门文章真实化

- 目标：
    - 侧栏“热门文章”按浏览量降序显示。
    - 点击后跳转正确文章详情。
- 验收标准：
    - 无固定假标题与假浏览量。

#### WEB-03 标签云真实化

- 目标：
    - 标签来源数据库，按关联文章数排序。
    - 点击标签可跳转到标签页或文章筛选页。
- 验收标准：
    - 标签数量和排序与数据库一致。

#### WEB-04 开源项目真实化（3 个项目）

- 目标：
    - 首页固定展示你维护的 3 个开源项目。
    - 展示名称、简介、仓库链接、Star/Fork（可选）。
- 实现建议：
    - MVP：先使用本地配置文件维护 3 个项目。
    - 增强：通过 GitHub API 定时拉取 Star/Fork 并缓存。
- 验收标准：
    - 三个卡片均可访问真实仓库，信息非占位。

---

### 4.3 About / Contact 页面（P0）

#### WEB-07 About 页面

- 建议模块：
    - 个人简介
    - 技术栈
    - 代表项目（可复用首页开源项目数据）
    - 联系方式入口
- 验收标准：
    - `/about` 可访问。
    - 移动端排版正常，关键信息完整。

#### WEB-08 Contact 页面

- 建议模块：
    - 联系方式（邮箱/GitHub/其他社媒）
    - 联系表单（姓名、邮箱、主题、内容）
    - 提交成功/失败反馈
- 验收标准：
    - `/contact` 可访问。
    - 表单字段校验与错误提示完整。
