# Blog 前台上线前优化任务清单

> 这是 `apps/blog` 当前优化周期的主任务文件，用于持久化保存 Blog 前台上线前已经确认的问题、当前可执行任务和后续专项拆分入口。

## 一、这份文件是做什么的

这不是最终整改方案，也不是某一个问题的详细技术设计。

这份文件的用途是：

- 保存当前阶段已经确认的 Blog 前台结论
- 把问题转成可以直接推进的任务
- 明确本轮优化边界和优先级
- 作为后续继续拆分专项文档时的单一入口

## 二、当前阶段说明

当前阶段仍然是：

- **Blog 前台上线前最终收口阶段**

这意味着：

- 已经完成一次针对 `apps/blog` 的整体盘点
- 已确认一批 `P0 / P1` 发布阻塞项
- 当前目标不是继续扩写泛化分析，而是进入任务化整改
- 后续更细的专项文档，必须基于这份清单继续拆分

## 三、当前优化约束

- 本轮优化只针对 `apps/blog` 现有系统，不做新功能扩展
- 当前先处理安全、数据正确性、构建基线和公开链路，再处理样式一致性与体验收口
- 未经确认，不重做后台整体认证与搜索架构，只修 Blog 前台当前可确认缺口
- 仓库级 CI、部署平台、monorepo 治理如需继续调整，应回到根目录 `docs/` 单列任务
- PWA 如果不满足 production 条件，优先缩 scope 或暂不上线
- 所有任务都必须以官网最佳实践和最终验证结果为准，不以“本地看起来能用”为完成标准

## 四、当前任务依据

本清单主要依据以下官方最佳实践整理：

- Next.js App Router：页面默认 `server-first`，`client component` 只下沉到叶子交互
- `next/font`：字体加载应以 `next/font/local` 或 `next/font` 为单一来源
- `next/image`：按 Next.js 16 当前 API 使用 `preload`、`loading`、`fetchPriority`
- Tailwind CSS 4：优先 CSS-first 配置；保留 legacy config 时必须显式接线
- Vercel Web Interface Guidelines：保留可见 focus、语义化交互、避免滥用 `transition: all`、照顾 reduced motion

## 七、当前明确可执行的任务：B. 安全与读写边界

### B-01 移除前台公开写文章接口

**任务目的**

封堵 Blog 前台当前最明显的公开写入风险，确保前台应用保持公开读侧边界。

**当前已发现的问题**

- `app/api/posts/route.ts` 暴露了可写 `POST` 接口
- 当前接口没有可确认的鉴权、授权、限流和后台边界控制
- 这会让公开站点存在匿名创建文章与标签的风险

**预期产物**

- Blog 前台不再暴露匿名可写文章接口
- 如果当前 UI 根本没有使用该接口，应直接移除而不是继续留空壳
- 如确实需要保留写能力，应把能力迁回后台受控链路

**验收标准**

- 匿名请求无法再通过 Blog 前台创建文章或标签
- 公开前台只保留读侧 contract

**当前状态**

- 已完成
- `about`、`photos`、`articles`、`snippets` 等页面私有实现已收进 `app/**/_components`、`app/**/_lib`

---

### B-02 统一已发布内容过滤规则

**任务目的**

让所有公开读链路使用同一套“已发布”定义，避免草稿泄漏和列表/详情不一致。

**当前已发现的问题**

- `lib/db.ts` 中多处查询只按 `published: true` 过滤
- 单篇详情页又额外排除了 `isDraft`
- 这会导致列表、标签、统计、sitemap、搜索等公开结果可能与详情页看到的集合不同

**预期产物**

- 抽出统一的“已发布内容”判定规则
- 以下读侧能力全部使用同一规则：
    - 文章列表
    - 文章数量统计
    - 标签统计
    - 首页聚合数据
    - sitemap
    - 搜索与搜索建议

**验收标准**

- 列表页、详情页、标签页、搜索和 sitemap 暴露的文章集合一致
- 草稿内容不会通过任何公开入口被带出

**当前状态**

- 已完成
- PWA 相关实现已从 `context/`、`components/pwa/`、`components/actions/pwa/` 收口到 `app/_features/pwa`

---

### B-03 修复标签筛选链路

**任务目的**

打通 `/tags -> /articles?tag=...` 的完整查询 contract，让标签页不再是表面可点、实际失效。

**当前已发现的问题**

- `/tags` 页会跳转到 `/articles?tag=...`
- 文章列表页没有真正按 `tag` 执行筛选
- 相关 API 对 `tag` 参数仍处于占位实现或直接返回空数据

**预期产物**

- 文章列表初始渲染支持 `tag` 过滤
- 相关分页、总数和后续请求都维持相同筛选条件
- 标签筛选与公开文章过滤规则保持一致

**验收标准**

- 从 `/tags` 点击任意标签后，可以看到真实的筛选结果
- 翻页或继续加载时筛选条件不丢失

**当前状态**

- 已完成
- 契约测试、单元测试、E2E 测试已统一放在 `tests/contracts`、`tests/unit`、`tests/e2e`

## 九、当前明确可执行的任务：D. 页面架构与性能

### D-01 缩小文章详情页 `client boundary`

**任务目的**

让文章详情页重新回到内容型页面应有的 `server-first` 结构，减少客户端 JS 和 hydration 压力。

**当前已发现的问题**

- 文章详情页当前由大范围 `client component` 包裹
- 文章只读内容渲染、目录、交互操作和统计组件被一起打进客户端
- 这与 Next.js 官方推荐的 App Router 页面组织方式不一致

**预期产物**

- 文章详情页拆成：
    - 服务端内容壳
    - 小粒度客户端交互岛
- 阅读正文、SEO、结构化内容优先保留在服务端

**验收标准**

- 页面交互不回退
- 文章页首屏和 hydration 成本明显低于当前实现

**当前状态**

- 已完成

---

### D-02 收敛文章详情页重复请求与热点大文件

**任务目的**

**预期产物**

- 合并可以合并的文章侧统计请求
- 开始把热点大文件按职责拆小
- 让后续功能改动不再总是集中压在几个超大文件上

**验收标准**

- 文章页客户端请求数量减少
- 至少完成第一轮热点文件职责收口

**当前状态**

- 已完成

### E-02 收敛字体系统与排版 token

**任务目的**

建立字体和排版的单一来源，避免当前重复声明、变量未挂载和未加载字体被直接使用的状态继续扩散。

**当前已发现的问题**

- `next/font/local` 与手写 `@font-face` 同时存在
- 字体变量没有真正挂到 `html` 或 `body`
- 页面上存在未实际加载的字体类名

**预期产物**

- 统一使用 `next/font` 作为字体加载入口
- 只保留必要的字体声明
- 排版类名只引用已经正式注册的字体 token

**验收标准**

- 全站字体来源单一且真实生效
- 不再出现“代码里写了字体名，但实际上没有加载”的情况

**当前状态**

- 已完成

---

### E-03 统一目录、文件命名与样式使用方式

**任务目的**

收敛当前文件命名、hooks 位置和样式写法不统一的问题，减少维护成本。

**当前已发现的问题**

- 组件文件、普通工具文件、hooks 目录存在混用命名风格
- 样式里同时存在 token 类和大量自由硬编码颜色/任意值
- 这会让后续新增代码继续扩散不一致

**预期产物**

- 组件文件统一遵守 `PascalCase.tsx`
- Hook 统一遵守 `useXxx.ts`
- 普通工具与配置模块优先使用 `kebab-case`
- 样式优先走统一 token，而不是继续堆叠任意值和零散色板

**验收标准**

- 新增和存量重点文件开始遵守统一命名规则
- 关键页面的颜色与排版写法明显更一致

**当前状态**

- 进行中

## 十一、当前明确可执行的任务：F. 可访问性与交互

### F-01 修复 focus、表单语义与媒体交互语义

**任务目的**

修复当前一批已确认的基础可访问性问题，避免站点在键盘可达性和表单语义上出现明显短板。

**验收标准**

- 登录、注册、评论等表单具备完整基础语义
- 媒体预览与关闭等操作不再依赖无语义容器

**当前状态**

- 已完成

### F-02 收敛动画与 reduced motion 策略

**任务目的**

把当前全局样式中的动画和过渡从“默认全开”收敛成更可控、可降级的交互体验。

**预期产物**

- 把广泛的 `transition: all` 改成更具体的过渡属性
- 给关键动画补齐 reduced motion 策略

**验收标准**

- 动画不会影响基础可用性
- 系统偏好减少动画时，页面仍能正常使用

**当前状态**

- 已完成

## 十二、当前明确可执行的任务：G. 图片、媒体与 PWA

### G-01 迁移图片与媒体加载策略到 Next 16 推荐写法

**任务目的**

收敛当前图片组件和媒体预览的加载策略，避免继续依赖已废弃 API 或保留 CLS 风险。

**当前已发现的问题**
